개발 의사결정 기록 (ADR)

ADR
001
상태 관리 : setState vs Provider
결정: setState 선택  |  근거: 단일 화면 위주 앱 특성상 전역 상태 관리 불필요. Provider 도입 시 오버엔지니어링 우려
ADR
002
로컬 저장소 : SQLite vs SharedPreferences
결정: SharedPreferences 선택  |  근거: 단어 데이터는 JSON 직렬화로 충분. SQLite는 복잡도 증가 대비 이점 불명확
ADR
003
단어 데이터 : API vs 로컬 하드코딩
결정: 로컬 하드코딩 선택  |  근거: 오프라인 환경 지원 필수. Free Dictionary API 응답 불안정성으로 UX 저하 우려
